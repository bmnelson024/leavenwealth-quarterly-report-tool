## Architecture overview

- **Pure client-side HTML/JS** — no framework, no bundler. Vanilla DOM + template-literal `innerHTML` rendering.
- **State** lives on a single top-level `state` object. `render()` re-draws the whole UI.
- **Persistence** via IndexedDB with a synchronous in-memory `_dbCache` mirror. Helpers: `dbSet`, `dbGet`, `saveProgress`, `loadProgress`, `restoreBackup`.
- **Three data-source modes** (selected via `selectSource(src)` at ~line 566):
  1. `appfolio` — AppFolio exports (rent rolls, income statement, leasing summary, renewal summary, trial balance, delinquency)
  2. `buildium` — Buildium exports (rent rolls rr1/rr2/rr3, income statement, leasing, bank, delinquency, capex detail, rent engine)
  3. `investment` — Multi-property investment-level reports (aggregates multiple properties)
- **File parsing:**
  - Excel → `XLSX.js` via CDN. `parseExcel`, `parseQuarterFromExcel`, `parsePriorQuarterFromExcel`, `xlsxToText`.
  - CSV/TSV → `parseDelim` (line ~660).
  - Month column detection uses module-scoped `parseColMonth`, `MO_ABBR_CAP`, `MO_ABBR_LOW`, `MONTH_COL_PAT`, `Q_MONTH_MAP`.
- **Photos & asset facts** keyed by `photoStorageKey(name)` / `normalizePropertyKey(name)` with NFKD diacritic stripping and a token-bounded `fuzzyKeyScore` fallback.
- **Narratives** auto-generated by `genNarratives` / `genInvestmentNarratives`; user edits tracked by `state.narrativesEdited` and `state.invNarrativesEdited` flags so re-processing doesn't clobber edited text.
- **XSS hygiene** via `escapeHtml` / `attrJs` helpers applied to every user-string interpolation site.
